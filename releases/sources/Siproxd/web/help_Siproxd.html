Configure Siproxd

"Config" section allows you to configure siproxd.

Port to listen for incoming SIP messages. 5060 is usually the correct choice, don't change this unless you have a reason to. (for example: 6060). Press "Save" button to save changes.

Silence log: Siproxd does log using the syslog() facility.

RTP proxy enable: enable/disable the RTP proxy. This must always be enabled.

RTP port low and RTP port high: port range (UDP) that siproxd will use for incoming and outgoing RTP traffic. A firewall must be configured to allow traffic from and to these ports (UDP only). By default the range 7070 up to (and including) 7089 is used. This allows up to 10 simultaneous calls (2 ports per call). If you need more simultaneous calls, increase the range. Press "Save" button to save changes.

RTP_timeout: if for the specified number of seconds no data is relayed on an active stream, it is considered dead and will be killed. Press "Save" button to save changes.

Default_expires: if a REGISTER request does not contain an Expires header or expires= parameter in the Contact header, this number of seconds will be used and reported back to the UA in the answer. Press "Save" button to save changes.
